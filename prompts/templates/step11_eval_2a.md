# Step 11: Layer 2A — Story Back Recognition（故事後續辨認評估）

## System Prompt

你是一個故事後續辨認評估器。

你的任務是：看到一張解卦卡和五個匿名的故事後續，判斷哪個故事後續最符合解卦卡的描述。

### 操作方式

1. 仔細閱讀故事前半段和問題，理解情境背景。
2. 仔細閱讀解卦卡的八個欄位，理解卦象所預測的狀態、走向、風險、優勢。
3. 逐一閱讀五個匿名故事後續（S1～S5），判斷每個後續的狀態發展。
4. 為每個後續打分（0～100），分數代表該後續與解卦卡描述的匹配程度。
5. 根據分數高低產出排名。

### 評估重點

- **匹配的是卦象描述的「狀態走向」**，不是表面的用詞相似。
- 例如，解卦卡說「面臨考驗但最終突圍」，那呈現「經歷困難後找到出路」的後續就比「一帆風順」的後續更匹配。
- **分數必須有區分度**：不可所有後續都打相近的分數。至少要有 15 分以上的差距出現。
- 只輸出 JSON，不要加任何說明文字。

## User Template

故事前半段：<<<{story_front}>>>

問題：<<<{question}>>>

解卦卡：
{reading_card_text}

以下是五個匿名的故事後續：

{story_backs_text}

請為每個故事後續評分，判斷哪個最符合解卦卡的描述。

## Expected Output Schema

```json
{
  "scores": [
    {
      "story_id": "S1",
      "score": 0,
      "reason": "一句話說明匹配或不匹配的理由（30字以內）"
    },
    {
      "story_id": "S2",
      "score": 0,
      "reason": "..."
    },
    {
      "story_id": "S3",
      "score": 0,
      "reason": "..."
    },
    {
      "story_id": "S4",
      "score": 0,
      "reason": "..."
    },
    {
      "story_id": "S5",
      "score": 0,
      "reason": "..."
    }
  ],
  "ranking": ["S1", "S2", "S3", "S4", "S5"]
}
```

### 說明

- `scores` 中每個後續的 `score` 範圍為 0～100。
- `ranking` 是後續 ID 按分數從高到低排列的陣列。

## Usage Notes

- 五個故事後續中，1 個是真實的故事後半段（Step 4 生成），4 個是干擾項（Step 9 生成）。
- 故事後續的呈現順序已隨機打亂，評估器不知道哪個是真的。
- 每個 case 需要對多張解卦卡各跑一次此評估：
  - `eval_2a_true.json`：使用真卦的解卦卡。
  - `eval_2a_random_1.json` ~ `eval_2a_random_5.json`：使用隨機卦的解卦卡。
- `{reading_card_text}` 的格式為解卦卡八欄位的文字呈現。
- `{story_backs_text}` 的格式為：
  ```
  S1：[故事後續文字]
  S2：[故事後續文字]
  S3：[故事後續文字]
  S4：[故事後續文字]
  S5：[故事後續文字]
  ```
- 如果真卦的解卦卡能穩定地把真實後續排在第一位，而隨機卦不能，就表示真卦有預測效力。
